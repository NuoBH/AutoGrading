## 发给同事或课程组的说明

这个工具适合在批量评阅视觉类或混合作业时使用。它的核心作用不是代替老师打分，而是减少反复下载、解压、打开文件、整理表格的时间。

比较适合的作业类型：

- 视频、图片、PDF、PPT、报告混合提交；
- 需要横向比较全班作品完成度；
- 需要最后整理成 Excel 分数和评语；
- 需要保留本地评阅记录，方便后续检查和修改。

不太适合的情况：

- 选择题、客观题或平台已经能自动批改的作业；
- 不允许使用本地辅助工具处理学生资料的课程；
- 需要工具直接把分数写回网页的场景。

## 知乎文章结构

标题：

高校老师如何更高效地批改超星 / 泛雅里的图片、视频、PDF 类作业？

结构：

1. 问题：混合作业不是难在“看不懂”，而是材料太分散；
2. 平台内批阅的局限：逐个打开、难横向比较、导出整理麻烦；
3. 本地辅助工作流：压缩包、评分标准、学生名单、evidence、作品总览、草稿评阅、确认转正、Excel；
4. 安全边界：不提交网页成绩、不替代老师判断、数据保存在本地；
5. 适合场景：视觉类、视频类、PDF / PPT / 报告混合类作业；
6. 不适合场景：客观题、平台自动批改已经足够的作业；
7. demo 截图和 repo 链接。

## 不建议使用的宣传说法

避免：

- 自动批改所有作业；
- 一键完成成绩录入；
- 不需要老师参与；
- 可直接替代人工评分；
- 自动绕过平台限制；
- 上传学生作业到云端智能批改。

更准确的说法：

- 本地辅助评阅；
- 自动整理材料；
- 生成建议分数和评语草稿；
- 老师确认后转为正式记录；
- 导出 Excel，方便检查和归档。
