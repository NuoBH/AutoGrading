# 泛雅 / 超星作业评阅助手

不用逐个解压、打开、截图、整理表格，把泛雅 / 超星里的图片、视频、PDF、PPT、报告和压缩包作业，整理成本地可批阅、可检查、可导出 Excel 的评阅流程。

它特别适合艺术、设计、影视、动画、数字媒体、三维、视觉传达等课程中的视觉类、视频类和混合作业。工具可以帮你整理学生材料、抽取视频帧、提取 PDF 文字或截取 PDF 页面、提取文档内容、生成作品总览图，并辅助写出建议分数和学生可读评语。

> 本工具只在本地生成评阅记录和表格，不会在泛雅 / 超星网页上提交、保存、发布或退回成绩。评分标准和最终结果都需要由使用者确认。

## 适合谁使用

如果你经常需要批阅视觉类、设计类或混合文件类作业，这个工具会比较有用。

它特别适合：

- 艺术、设计、动画、影视、数字媒体、三维、视觉传达等课程作业；
- 学生提交中包含图片、视频、PDF、PPT、报告、源文件或压缩包；
- 需要快速横向比较全班作品完成度、视觉效果和提交质量；
- 希望批量生成较自然、可直接给学生看的评语；
- 希望把最终分数和评语整理成 Excel 表格。

如果作业主要是纯文字文档，也可以使用；只是它通常不会默认走“作品总览图”快速分档。工具会先为每个学生尽量提取 docx、pptx、txt、markdown 和可选中文本型 PDF 的内容，再合并成一个全班文字评阅包，方便先整体阅读和分档；如果文本不足、乱码或需要看版式，再打开原始文件补看。

## 它能帮你做什么

它帮你把分散、混乱、格式各异的学生附件，变成可批量浏览、可复核、可导出表格的本地评阅流程。

- 整理学生名单和附件，减少手动解压、查找、重命名。
- 从视频抽帧，从 PDF 提取文字或页面，从 Word / PPT / 文本文件提取可读内容。
- 为视觉类作业生成作品总览图，为文档类作业生成全班文字评阅包。
- 辅助生成建议分数和学生可读评语。
- 导出 Excel，并保留本地评阅记录，方便检查、修改和续评。

## 最终会得到什么

一次完整评阅通常会生成这些结果：

```text
rubrics/   评分标准文件
result/    每个学生的建议分数和评语记录
outputs/   最终 Excel 成绩表
tmp/       临时处理文件、抽帧图片、PDF 文字/截图和会话状态
```

常见的最终 Excel 可以包含：

- 学生姓名
- 学号 / 编号
- 分数
- 评语

如果只需要“学生姓名 / 分数 / 评语”三列，也可以直接让工具按这个格式导出。


## 获取这个项目

### 方式一：使用 Git 克隆

在终端、PowerShell 或 Git Bash 中运行：

```bash
git clone https://github.com/NuoBH/AutoGrading.git
cd AutoGrading
```

### 方式二：直接下载 ZIP

如果你不熟悉 Git，也可以直接下载仓库 ZIP：

1. 打开 GitHub 仓库页面。
2. 点击绿色的 **Code** 按钮。
3. 选择 **Download ZIP**。
4. 解压到本地文件夹。
5. 用终端、PowerShell、VS Code 或 Codex 等本地 AI 助手打开解压后的项目根目录。

解压后请确认你打开的是项目根目录，而不是外层多套了一层的空文件夹。


## 使用前准备

目前最推荐在 **Windows** 上使用。第一次下载仓库后，建议先在项目根目录运行：

```powershell
node tools/fanya/scripts/setup-environment.cjs
```

它会安全地做三件事：

1. 创建本地工作文件夹：`tmp/`、`tmp/bundle/`、`tmp/session/`、`rubrics/`、`result/`、`outputs/`；
2. 检查本机是否能找到必要工具；
3. 打印缺少依赖时的安装建议。

默认情况下，它**不会自动安装系统软件**，也不会改你的浏览器登录状态。

### 必须安装

* **Node.js / npm**：运行本项目脚本需要。
* **Chrome**：网页登录和网页批阅模式需要。
* **browser-act**：当前工作流使用它进行浏览器导航、读取作业说明、抓取网页学生名单；因此是必需依赖。
* **Codex 或其他支持本地文件操作的 AI 助手**：用于按照评阅流程读取作业、整理材料、生成评分标准、写入评阅记录和导出 Excel。
* **泛雅 / 超星账号权限**：需要能够访问对应课程、作业和学生提交内容。

### 推荐安装

* **Git**：用于克隆和更新仓库；如果直接下载 ZIP，可以暂时不用。
* **ffmpeg + ffprobe**：视频作业抽帧需要；`ffprobe` 可以让抽帧更准确。
* **7-Zip**：处理 `.zip`、`.7z`、`.rar` 等压缩包更稳定。
* **Poppler / pdftoppm / pdftotext**：把 PDF 页面渲染成图片、从文本型 PDF 中提取文字需要。
* **Python 3 + Pillow**：生成 PNG 版作品总览图需要。

### 常用初始化命令

只检查环境：

```powershell
node tools/fanya/scripts/doctor.cjs
```

只创建项目文件夹：

```powershell
node tools/fanya/scripts/setup-environment.cjs --prepare
```

安装 Python Pillow：

```powershell
node tools/fanya/scripts/setup-environment.cjs --install-python-packages
```

系统软件如 Node.js、Chrome、browser-act、ffmpeg、7-Zip、Poppler 不会被脚本自动静默安装。请按 `setup-environment.cjs` 打印的提示手动安装，或让 AI 助手在你确认后再执行安装命令。


## 快速开始

### 方式一：已有全班作业压缩包

这是最推荐的方式。

1. 从泛雅 / 超星下载全班作业压缩包。
2. 把压缩包放入：

   ```text
   tmp/bundle/
   ```

3. 在 Codex 或其他本地 AI 助手中输入：

   ```text
   开始泛雅评阅
   ```

4. 选择“压缩包批阅模式”。
5. 按提示确认课程、作业、评分标准和评阅记录。
6. 等工具解压作业并生成真实学生名单。
7. 确认是否有不评阅的学生。
8. 等工具准备作业材料。
9. 视觉类或混合作业可以先查看作品总览图和草稿评阅摘要；文档类作业可以先查看全班文字评阅包，再按需要补看个别学生原始材料。
10. 确认草稿可以转为正式评阅后，导出 Excel。


### 压缩包内的推荐文件夹格式

压缩包内推荐使用“每个学生一个文件夹”的结构。每个学生文件夹的名称建议使用：

```text
学号_姓名
```

每个学生自己的作业附件都放在对应的学生文件夹里。

推荐结构示例：

```text
parent/
├─ 20240001_张三/
│  ├─ 作业文件.pdf
│  ├─ 作品截图.png
│  └─ 视频作品.mp4
├─ 20240002_李四/
│  ├─ final.pptx
│  └─ 素材说明.docx
└─ student-003_StudentName/
   ├─ render.jpg
   ├─ report.pdf
   └─ source.zip
```

也就是说，压缩包解压后应当能看到一个总文件夹，里面按学生分成多个子文件夹；每个子文件夹里放该学生提交的所有附件。



### 方式二：从网页逐个批阅

如果没有全班压缩包，或者必须从网页逐个打开学生提交，可以使用网页批阅模式。

1. 在 Codex 或其他本地 AI 助手中输入：

   ```text
   开始泛雅评阅
   ```

2. 选择“网页批阅模式”。
3. 打开并登录超星页面：<https://i.chaoxing.com/>
4. 按提示进入对应课程和作业。
5. 确认评分标准和评阅记录。
6. 让工具抓取真实学生名单和网页状态。
7. 确认是否有不评阅的学生。
8. 工具会逐个学生下载或查看附件，并把结果写入本地记录。

网页批阅模式更慢、更依赖浏览器状态，因为需要逐个学生打开或下载附件。如果能下载全班作业压缩包，通常更推荐压缩包批阅模式。

## 你主要负责确认，工具负责重复劳动

你通常只需要做几个关键确认：

- 放入作业压缩包，或登录泛雅 / 超星网页。
- 确认课程、作业、评分标准和评阅记录文件。
- 确认是否有不评阅的学生，是否同步网页上“已完成”的学生状态。
- 确认基于作品总览图或全班文字评阅包生成的草稿评阅是否可以转为正式评阅。
- 确认是否导出 Excel，是否清理临时文件。

工具会负责：

- 解压、下载或准备学生作业材料。
- 整理真实学生名单和当前进度。
- 生成视频帧、PDF 文字或页面、文档文本、每个学生的文本评阅材料、全班文字评阅包和作品总览图。
- 按评分标准优先查看关键材料，辅助写出建议分数和学生可读评语。
- 写入本地评阅记录，导出 Excel，并在你确认后清理临时文件。

## 文件放置和输出位置

### 作业压缩包

把从泛雅 / 超星下载的作业总包放在：

```text
tmp/bundle/
```

不要删除 `tmp/bundle/` 文件夹本身。评阅完成后，可以让工具删除已经处理完的压缩包和解压目录，但不要手动清空整个 `tmp/`。

### 评分标准文件

评分标准文件会放在：

```text
rubrics/
```

通常会按课程分文件夹，例如：

```text
rubrics/<课程名>/<作业名>-rubric.cjs
```

你不需要手动写这个文件。工具会根据作业要求生成评分标准，并让你确认。确认后，后续同一个作业可以复用。

### 评阅记录文件

每个学生的建议分数和评语会写入：

```text
result/
```

通常文件名类似：

```text
<课程名>-作业评价汇总-<日期>.cjs
```

这是最重要的本地评阅记录。导出 Excel、继续评阅、恢复进度都会用到它。

### 最终 Excel 表格

导出的 Excel 默认放在：

```text
outputs/
```

一个评阅记录文件可以导出一个 Excel 工作簿。如果里面包含多个作业，每个作业会放在不同工作表里。

### 临时工作文件

工具运行时会产生临时文件，主要在：

```text
tmp/work-*/
tmp/session/
```

这些文件用于解压作业、保存当前进度、存放抽帧图片、PDF 文字/截图、文档提取内容等。评阅没有完成前不要随便删除。


## 其他说明

### 作品总览图是什么

作品总览图会把每个学生最关键的图片、视频帧或 PDF 页面拼在一起，形成一张或多张大图。

它的作用是：

- 快速看出全班整体水平；
- 更容易控制分数分布；
- 找出明显优秀、明显敷衍、材料缺失或需要补看的学生；
- 减少逐个打开所有文件的时间。

工具会保留总览图的原始记录，也可以生成更适合查看的 PNG 预览。如果学生很多，或者每个学生需要多张图 / 多帧视频，也可以分页生成多张总览图。

纯文字或长文档作业通常不默认使用作品总览图。它会使用下面的“全班文字评阅包”流程。

### 文字 / 文档类作业怎么批阅

对于主要由 docx、pptx、txt、markdown、文本型 PDF、报告或反思文档组成的作业，工具会优先走文字证据流程：

1. 先为每个学生生成自己的 `review-text.md`，尽量汇总该学生可提取的文字内容。
2. 再把未完成、未跳过学生的文本材料合并成 `assignment-review-text.md`，也就是全班文字评阅包。
3. 先阅读全班文字评阅包，做第一轮分档和草稿评阅。
4. 对文本缺失、乱码、被截断、边界分、明显高分、明显低分或材料冲突的学生，再打开该学生的原始文件补看。

如果是混合型作业，例如报告 + 图片、PPT + 视频、PDF 页面 + 设计图，工具会同时使用全班文字评阅包和作品总览图：文字包用于看报告、说明和反思，作品总览图用于看画面、版式、视频帧或页面效果。两者都是评阅证据，最终仍要按评分标准和用户确认转成正式评阅记录。

### 快速批阅如何变成正式结果

压缩包批阅模式会按评分标准选择快速批阅路径：视觉 / 视频 / 混合作业通常用作品总览图，纯文字 / 文档作业通常用全班文字评阅包。快速批阅不会直接写成最终成绩。

完整流程是：

1. 先根据评分标准挑出最重要的材料。
2. 视觉类作业生成作品总览图；文档类作业准备每个学生自己的文本评阅材料，并可生成全班文字评阅包。
3. 写第一轮草稿评阅，用来先检查分数分布和评语质量。
4. 草稿评语也必须是学生可见的正式语气，不能出现“草稿”“复核”“当前分数”“正式评分应”等内部流程话术。
5. 对异常、边界分、高分候选、低分候选再补看材料。
6. 工具先做一次转正式预检查，并给出检查摘要。
7. 用户确认后，草稿才会转为正式评阅记录。

这样可以在保证基本公平和准确的前提下，减少大量重复打开文件的时间。

### 评分标准和评语质量

评分标准文件记录这个作业的打分规则，包括：

- 作业满分；
- 各分数段大致含义；
- 本次作业重点看什么；
- 哪些文件优先查看；
- 是否适合使用快速批阅；
- 评语应该多长、要不要更个性化。

评分标准生成后，工具必须先给你确认。你确认之后，才会开始评学生。

正式评语会尽量保持具体、自然、可直接给学生看。默认要求是：

- 通常写 2-3 句；如果作业复杂，可以写 3-5 句；
- 点出作品主题、画面特点、完成度或材料表现；
- 给出一个具体改进方向；
- 不写后台流程话术，例如“可保持当前分数”“分数应偏低”“高分候选”“需复核”“正式评阅时再看”等；
- 不为了省事写一批几乎一样的模板评语。

### 安全边界

这个工具的设计目标是辅助本地评阅，而不是代替你在网页上直接操作成绩。

请注意：

- 工具只做本地评阅，不会把分数提交到网页；
- 不要把账号密码写进项目文件；
- 不要把学生真实信息写进脚本、测试、流程文档或 README；
- 本地评阅记录和最终 Excel 会包含学生信息，请按教学资料妥善保存；
- 如果网页弹出提交、保存、发布、退回、导入成绩等按钮，不要让工具点击。

### 强制流程保障

为了避免误评、漏评或使用假名单，每次开始或继续评阅时，工具都必须先检查当前任务状态。

正式评阅前必须满足：

1. 已进入评阅向导；
2. 已确认评阅模式、评分标准、评阅记录和学生名单；
3. 评分标准已经由用户确认；
4. 学生名单来自真实压缩包或真实网页列表；
5. 没有使用示例名单、占位名单或手写假名单；
6. 已确认是否有不评阅的学生；
7. 向导状态已经明确允许开始评阅。

如果 AI 助手在没有完成这些步骤前就想直接评学生、生成总览图或写结果，请让它停下，重新从：

```text
开始泛雅评阅
```

或：

```text
继续评阅
```

进入流程。

### 学生名单和评阅进度

工具必须先生成真实学生名单，才能开始评阅。

- 压缩包批阅模式：学生名单来自解压后的学生文件夹；
- 网页批阅模式：学生名单来自网页作业批阅列表；
- 学生名单里的“导入时状态”只是当时看到的网页或压缩包状态，不代表最终评阅进度；
- 最终谁已评、谁跳过、谁需要人工复查，以 `result/` 里的评阅记录和当前任务进度为准；
- 不能用示例名单、占位名单或手写假名单进入正式评阅。

### 评阅完成后怎么清理

一个作业全部评完、Excel 也导出后，可以让 Codex 或其他本地 AI 助手清理临时文件：

```text
这个作业已经评完了，帮我清理临时文件
```

清理时会遵守：

- 可以删除这个作业对应的解压目录；
- 可以删除这个作业已经处理完的压缩包；
- 可以清空当前任务进度；
- 不删除 `tmp/bundle/` 文件夹本身；
- 不删除其他还没评阅的压缩包；
- 不删除 `rubrics/`、`result/`、`outputs/` 里的正式文件。

如果只是使用这个工具，一般不需要看开发文档。
